485=head1 DESCRIPTION
486
487This component is a method to simplify the SSLification of a socket before it is passed
488to a L<POE::Wheel::ReadWrite> wheel in your application.
489
490=head2 Client usage
491
492 # Import the module
493 use POE::Component::SSLify qw( Client_SSLify );
494
495 # Create a normal SocketFactory wheel and connect to a SSL-enabled server
496 my $factory = POE::Wheel::SocketFactory->new;
497
498 # Time passes, SocketFactory gives you a socket when it connects in SuccessEvent
499 # Convert the socket into a SSL socket POE can communicate with
500 my $socket = shift;
501 eval { $socket = Client_SSLify( $socket ) };
502 if ( $@ ) {
503 # Unable to SSLify it...
504 }
505
506 # Now, hand it off to ReadWrite
507 my $rw = POE::Wheel::ReadWrite->new(
508 Handle => $socket,
509 # other options as usual
510 );
511
512=head2 Server usage
513
514 # !!! Make sure you have a public key + certificate
515 # excellent howto: http://www.akadia.com/services/ssh_test_certificate.html
516
517 # Import the module
518 use POE::Component::SSLify qw( Server_SSLify SSLify_Options );
519
520 # Set the key + certificate file
521 eval { SSLify_Options( 'server.key', 'server.crt' ) };
522 if ( $@ ) {
523 # Unable to load key or certificate file...
524 }
525
526 # Create a normal SocketFactory wheel to listen for connections
527 my $factory = POE::Wheel::SocketFactory->new;
528
529 # Time passes, SocketFactory gives you a socket when it gets a connection in SuccessEvent
530 # Convert the socket into a SSL socket POE can communicate with
531 my $socket = shift;
532 eval { $socket = Server_SSLify( $socket ) };
533 if ( $@ ) {
534 # Unable to SSLify it...
535 }
536
537 # Now, hand it off to ReadWrite
538 my $rw = POE::Wheel::ReadWrite->new(
539 Handle => $socket,
540 # other options as usual

543=head1 NOTES
544
545=head2 Socket methods doesn't work
546
547The new socket this module gives you actually is tied socket magic, so you cannot do stuff like
548getpeername() or getsockname(). The only way to do it is to use L</SSLify_GetSocket> and then operate on
549the socket it returns.
550
551=head2 Dying everywhere...
552
553This module will die() if Net::SSLeay could not be loaded or it is not the version we want. So, it is recommended
554that you check for errors and not use SSL, like so:
555
556 eval { use POE::Component::SSLify };
557 if ( $@ ) {
558 $sslavailable = 0;
559 } else {
560 $sslavailable = 1;
561 }
562
563 # Make socket SSL!
564 if ( $sslavailable ) {
565 eval { $socket = POE::Component::SSLify::Client_SSLify( $socket ) };
566 if ( $@ ) {
567 # Unable to SSLify the socket...
568 }
569 }

571=head3 $IGNORE_SSL_ERRORS
572
573As of SSLify v1.003 you can override this variable to temporarily ignore some SSL errors. This is useful if you are doing crazy things
574with the underlying Net::SSLeay stuff and don't want to die. However, it won't ignore all errors as some is still considered fatal.
575Here's an example:
576
577 {
578 local $POE::Component::SSLify::IGNORE_SSL_ERRORS=1;
579 my $ctx = SSLify_CreateContext(...);
580 #Some more stuff
581 }

583=head2 OpenSSL functions
584
585Theoretically you can do anything that Net::SSLeay exports from the OpenSSL libs on the socket. However, I have not tested every
586possible function against SSLify, so use them carefully!
587
588=head3 Net::SSLeay::renegotiate
589
590This function has been tested ( it's in C<t/2_renegotiate_client.t> ) but it doesn't work on FreeBSD! I tracked it down to this security
591advisory: L<http://security.freebsd.org/advisories/FreeBSD-SA-09:15.ssl.asc> which explains it in detail. The test will skip this function
592if it detects that you're on a broken system. However, if you have the updated OpenSSL library that fixes this you can use it.
593
594NOTE: Calling this means the callback function you passed in L</Client_SSLify> or L</Server_SSLify> will not fire! If you need this
595please let me know and we can come up with a way to make it work.
596
597=head2 Upgrading a non-ssl socket to SSL
598
599You can have a normal plaintext socket, and convert it to SSL anytime. Just keep in mind that the client and the server must agree to sslify
600at the same time, or they will be waiting on each other forever! See C<t/3_upgrade.t> for an example of how this works.
601
602=head2 Downgrading a SSL socket to non-ssl
603
604As of now this is unsupported. If you need this feature please let us know and we'll work on it together!
605
606=head2 MSWin32 is not supported
607
608This module doesn't work on MSWin32 platforms at all ( XP, Vista, 7, etc ) because of some weird underlying fd issues. Since I'm not a windows
609developer, I'm unable to fix this. However, it seems like Cygwin on MSWin32 works just fine! Please help me fix this if you can, thanks!

615and you would need to explicitly enable it.
